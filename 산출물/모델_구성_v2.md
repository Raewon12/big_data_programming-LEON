|---|---|---|---|
| ① | TF-IDF+BERT 하이브리드를 쓴다고 했는데 **구체적으로 어떻게 결합할지 불명확** | 🔴 구조 | Late Fusion으로 명시 |

## 3. 피드백별 해결 상세

### ① 하이브리드 구체화 — Late Fusion

두 모델을 **각자 따로 학습**시킨 뒤, **예측 확률만 가중 평균**한다.

```
P_final = α · P(TF-IDF + LR) + (1 − α) · P(BERT + MLP)

α ∈ {0.0, 0.1, 0.2, ..., 1.0}  → 검증셋(val.csv)에서 최적 α 탐색
최종 예측 = argmax(P_final)
```

- **왜 LR인가**: SVM(LinearSVC)은 확률 출력이 까다로움 → 확률 출력이 깔끔한 LR 사용
- **왜 Late Fusion인가**: 차원이 다른 두 신호(TF-IDF 10000차원 ↔ BERT 768차원)를 직접 합치지 않고, 결정(확률) 단계에서 결합 → 차원 충돌 없음
- 학계에서 검증된 정식 앙상블 기법 → "이상한 방식" 비판 차단

### ② 토큰 수 근거 위치 조정

| 위치 | 처리 |
|---|---|
| EDA 분석 (PPT p12) | **유지** — 데이터 관찰·입력 잘림 판단 근거로는 정당 |
| 모델 설계 (PPT p15, p18) | **삭제** — 분류 단계에서는 의미 없음 |

- BERT는 입력을 512로 truncation 후 `[CLS]` 토큰 → **768차원 벡터 1개**만 추출
- 분류기 입장에서는 토큰 수가 사라지고 "고정 768차원 입력"일 뿐
- 따라서 "512 토큰 한계"를 모델 설계의 핵심 근거로 쓰는 것은 부적절

### ③ "잘 안 쓰는 방식" → Late Fusion으로 정당화

| 방식 | 문제점 | 채택 |
|---|---|---|
| Early Fusion (concat) | TF-IDF 10000차원이 BERT 768차원을 압도 → BERT 신호 묻힘 | ❌ |
| **Late Fusion (확률 가중합)** | 차원 충돌 없음, 각 모델이 잘하는 클래스에서 상호 보완 | ⭕ |

명분: **"어휘 신호(TF-IDF)와 문맥 신호(BERT)를 결정 단계에서 결합"**

### ④ 딥러닝 비교군 추가 — TextCNN

- 기존: TF-IDF / BERT만 → 비교군 빈약
- 추가: **TextCNN** (학습형 임베딩 기반, NLP 분류 클래식)
- 결과적으로 **표현 방식 3종** 비교 구도 완성:

| 표현 방식 | 대표 모델 |
|---|---|
| 희소 빈도 | TF-IDF + LR/SVM |
| 학습 임베딩 | TextCNN |
| 사전학습 문맥 | KLUE-BERT(frozen) + MLP |

## 5. BERT 활용 방식 (PPT p18 교체용)

| 방식 | 학습 비용 | 본 프로젝트 |
|---|---|---|
| Fine-tuning | 매우 높음 | ❌ 미채택 (GPU 제약 + 공정 비교 명분) |
| **Frozen Feature Extractor** | **낮음** | ⭕ **채택** |

**선정 이유** (수정):
1. 사전학습 가중치를 그대로 사용 → 어떤 표현이 더 좋은지 **공정 비교** (fine-tuning으로 BERT만 유리해지는 것 방지)
2. 임베딩 1회 추출 후 `.npy` 캐싱 → 실험 반복 속도 ↑
3. ~~512 토큰 한계~~ → 이 근거는 제거 (피드백 ② 반영)

## 6. 작업 단위 (노트북 구성)

```
bert/
├── extract_embeddings.ipynb   # KLUE-BERT 임베딩 1회 추출 → .npy 캐싱 (김홍근)
└── bert_mlp.ipynb             # 임베딩 로드 → MLP 학습·평가 (김홍근)
textcnn/
└── textcnn.ipynb              # 토큰화 → TextCNN 학습·평가 (미정)
hybrid/
└── late_fusion.ipynb          # LR 확률 + BERT 확률 가중합, α 탐색 (김홍근)
analysis/
└── final_comparison.ipynb     # 5개 모델 통합 비교표·그래프 (정래원)
```

**공통 규칙**: 모든 입력은 **결합**(`jdgmn + ' ' + summ_pass`) 사용 (1차 실험 베스트 입력)
